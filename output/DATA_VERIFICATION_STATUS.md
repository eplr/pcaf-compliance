# PCAF® Compliance — Data Verification Status

> Date: 8 April 2026
> Repository: https://github.com/eplr/pcaf-compliance

---

## Context

Following Axylia's feedback identifying errors in the assessment data, a full audit was conducted. The root cause was identified: the source data feeding the scoring tool had been generated by automated extraction from sustainability reports, without systematic human verification. The audit found a 27% error rate on the 3 client-verified institutions alone.

A new approach was implemented: **full reading of the source report** (not regex sampling), with every score justified by a verbatim quote and line/page reference.

## What was done

1. **Root cause analysis**: identified that all source data (JSON files) was AI-generated from reports without human verification. 27% error rate measured on 3 test institutions.

2. **New approach implemented**: XHTML-to-clean-text conversion (121 MB → 1.86 MB) enables full report reading. Applied to ING as proof of concept — corrected 6 out of 8 scores vs. the auto-scorer.

3. **Flagging**: all 231 auto-generated scores now prefixed with `[UNVERIFIED]` in the CSV. No data has been invented or estimated.

4. **Methodology updated**: `methodology/PCAF_Methodologie_V3_FR.md` now documents the full-reading approach as the required method.

---

## What remains to be done

To verify the remaining 21 institutions, each requires:

1. Obtain the source report (PDF or ESEF XHTML if available)
2. Convert to clean text (for XHTML) or use existing extracted text (for PDF)
3. Read the sustainability/climate sections in full
4. For each of the 8–11 criteria: find the verbatim data, apply the scoring grid, document the evidence
5. Produce an assessment report (`output/{INSTITUTION}_PCAF_Compliance_Assessment.md`)

Estimated effort per institution: depends on report length and PCAF disclosure quality. ING (the most detailed report) took ~1 hour of reading and scoring.
